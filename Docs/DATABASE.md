# DATABASE — SysCalls (RSC)

Спецификация формата трёх слоёв БД и финального `canonical.toml`.

## Обзор файлов БД

```
db/
├── auto/                          # Layer 1: ground truth (from ntdll.pdb)
│   ├── 10_19045_6466.toml         # one per Windows build
│   ├── 11_22631_2514.toml
│   └── 11_26100_2314.toml
├── phnt/
│   └── phnt.toml                  # Layer 2: typed signatures from phnt
├── overrides.toml                 # Layer 3: manual fixes (only hand-edited file)
└── canonical.toml                 # Generated: merged result (derived)
```

## 1. Auto Layer — `db/auto/<build>.toml`

Создаётся `rsc-collector`. **Редактировать вручную запрещено** (перезаписывается).

### Схема

```toml
# Auto-collected ntdll syscall snapshot
# DO NOT EDIT — regenerated by rsc-collector
# Schema version: 1

[meta]
schema_version = 1
build_id = "11_26100_2314"          # {Major}_{Build}_{UBR}
windows_version = "11 24H2"
collected_at = "2026-04-24T12:34:56Z"
collector_version = "0.1.0"

[meta.ntdll_x64]
path = "C:\\Windows\\System32\\ntdll.dll"
file_size = 2203648
pdb_name = "ntdll.pdb"
pdb_guid = "ABCDEF01-2345-6789-ABCD-EF0123456789"
pdb_age = 1
pdb_sha256 = "deadbeef..."          # of the downloaded PDB

[meta.ntdll_x86]                    # optional, only on x64 Windows
path = "C:\\Windows\\SysWOW64\\ntdll.dll"
file_size = 1847296
pdb_name = "ntdll.pdb"
pdb_guid = "..."
pdb_age = 1
pdb_sha256 = "..."

# Each syscall is a [[syscall]] entry
[[syscall]]
name = "NtAllocateVirtualMemory"
# SSN discovered per arch (may differ)
ssn_x64 = 24
ssn_x86 = 24
# Arity discovered per arch
arity_x64 = 6
arity_x86 = 6
# RVA within respective ntdll
rva_x64 = 0x1A2B30
rva_x86 = 0x5C8F0
# Raw stub bytes (first 20 bytes) for debugging / verification
stub_x64 = "4C8BD1B8180000000F05C3CCCCCCCC"
stub_x86 = "B8180000BA5076770077D280000000"

[[syscall]]
name = "NtFreeVirtualMemory"
ssn_x64 = 30
ssn_x86 = 30
arity_x64 = 4
arity_x86 = 4
rva_x64 = 0x1A2E20
rva_x86 = 0x5CA10
stub_x64 = "..."
stub_x86 = "..."

# ... ~500 entries ...
```

### Правила
- **Порядок**: syscalls отсортированы по `name` (стабильный diff)
- **Имена**: начинаются с `Nt` (не `Zw`) — в ntdll они совпадают, выбираем `Nt` для консистентности
- **SSN**: `u16`, всегда известен (иначе функция пропускается с warning)
- **Arity**: `u8`, x86 точно, x64 heuristic
- **RVA/stub**: для debugging и drift-check, не используется в runtime

### Идемпотентность
- Пересборка того же билда = тот же файл (bit-identical при одинаковом PDB)
- `collected_at` — только у вновь сгенерированных

## 2. Phnt Layer — `db/phnt/phnt.toml`

Создаётся `rsc-types`. **Редактировать вручную запрещено** (регенерируется из phnt).

### Схема

```toml
# Phnt-derived typed signatures
# Source: vendor/phnt @ <commit-hash>
# DO NOT EDIT — regenerated by rsc-types

[meta]
schema_version = 1
phnt_commit = "abc123def456..."
phnt_revision = "phnt-2026-04-01"
parsed_at = "2026-04-24T12:34:56Z"
types_version = "0.1.0"

# Canonical type mapping (phnt → rsc types)
[type_mapping]
"PHANDLE" = "*mut HANDLE"
"PVOID" = "*mut c_void"
"PULONG" = "*mut u32"
"PSIZE_T" = "*mut usize"
# ... hundreds of mappings

# Per-function signature
[[function]]
name = "NtAllocateVirtualMemory"
return_type = "NTSTATUS"
# Parameters in declaration order
params = [
    { name = "ProcessHandle",    type = "HANDLE",     direction = "in" },
    { name = "BaseAddress",      type = "*mut PVOID", direction = "inout" },
    { name = "ZeroBits",         type = "ULONG_PTR",  direction = "in" },
    { name = "RegionSize",       type = "*mut SIZE_T", direction = "inout" },
    { name = "AllocationType",   type = "u32",        direction = "in" },
    { name = "Protect",          type = "u32",        direction = "in" },
]
# Minimum phnt version this signature is valid for
min_phnt_version = "PHNT_WIN7"
# Windows availability (from phnt macros)
available_since = "WIN_7"       # or null if universal
deprecated_since = null

[[function]]
name = "NtClose"
return_type = "NTSTATUS"
params = [
    { name = "Handle", type = "HANDLE", direction = "in" },
]
available_since = null          # present since NT

# ...
```

### Правила
- Порядок: по `name`
- `params.direction`: `in` / `out` / `inout` — извлекается из `_In_` / `_Out_` / `_Inout_` SAL annotations phnt
- `min_phnt_version`: соответствует `#if (PHNT_VERSION >= ...)` gatings
- Функция может отсутствовать в phnt → в canonical получит opaque signature

## 3. Overrides Layer — `db/overrides.toml`

**Единственный файл, редактируемый вручную**. Priority наивысший.

### Назначение

- Исправить bug в phnt signature
- Добавить syscall которого нет ни в phnt, ни в auto (кастомный / недокументированный)
- Исключить syscall из сборки (exclude list)
- Переопределить arity если collector ошибся
- Отметить функцию как deprecated / unstable

### Схема

```toml
# Manual overrides — highest priority in merge
# Hand-edit as needed, commit with explanation

[meta]
schema_version = 1

# ─── Fix phnt signature bug ────────────────────────────────
# Reason: phnt declares Param3 as ULONG, but on Win11 24H2
# it's actually ULONG_PTR. Verified via WinDbg.
[[override]]
kind = "fix_signature"
name = "NtSomeFunction"
return_type = "NTSTATUS"
params = [
    { name = "Handle",  type = "HANDLE",    direction = "in" },
    { name = "Buffer",  type = "*mut PVOID", direction = "out" },
    { name = "Size",    type = "ULONG_PTR", direction = "in" },  # was ULONG in phnt
]
reason = "Win11 24H2 changed Param3 from ULONG to ULONG_PTR"
verified_on = ["11_26100_2314"]

# ─── Add custom syscall ────────────────────────────────────
[[override]]
kind = "add"
name = "NtCustomUndocumentedFunction"
return_type = "NTSTATUS"
ssn_x64 = 0x1F7
ssn_x86 = 0x1F7
arity = 3
params = [
    { name = "Arg1", type = "HANDLE", direction = "in" },
    { name = "Arg2", type = "PVOID", direction = "in" },
    { name = "Arg3", type = "u32",   direction = "in" },
]
reason = "Undocumented syscall discovered via kernel RE, see dev notes"

# ─── Exclude from build ────────────────────────────────────
[[override]]
kind = "exclude"
name = "NtUnusedDeprecatedCall"
reason = "Deprecated since Win10; not needed in this project"

# ─── Fix arity ─────────────────────────────────────────────
[[override]]
kind = "fix_arity"
name = "NtSomeOther"
arity_x64 = 7
arity_x86 = 7
reason = "Collector heuristic returned 6, actually 7 (verified in disassembly)"
```

### Правила
- `kind` определяет тип override: `fix_signature`, `fix_arity`, `add`, `exclude`, `fix_ssn`
- `reason` — **обязательное поле**, без объяснения PR не принимается
- `verified_on` — список build IDs где проверено
- Новые `kind` добавляются по мере необходимости

## 4. Canonical DB — `db/canonical.toml`

**Derived artifact**, генерируется `rsc merge`. Читается `rsc-runtime/build.rs`.

### Схема

```toml
# Canonical syscall database — merged source of truth
# Generated by `rsc merge` — DO NOT EDIT

[meta]
schema_version = 1
generated_at = "2026-04-24T13:00:00Z"
merge_tool_version = "0.1.0"
baseline_build = "11_26100_2314"      # which auto/*.toml was used as base
phnt_commit = "abc123..."
phnt_revision = "phnt-2026-04-01"
overrides_hash = "sha256:..."

[meta.layers_merged]
auto = "db/auto/11_26100_2314.toml"
phnt = "db/phnt/phnt.toml"
overrides = "db/overrides.toml"

# All syscalls, alphabetical
[[syscall]]
name = "NtAllocateVirtualMemory"
# Sources: which layers contributed
sources = ["auto", "phnt"]
# From auto
ssn_x64 = 24
ssn_x86 = 24
arity_x64 = 6
arity_x86 = 6
rva_x64 = 0x1A2B30
rva_x86 = 0x5C8F0
# From phnt (or overrides)
return_type = "NTSTATUS"
params = [
    { name = "ProcessHandle",    type = "HANDLE",     direction = "in" },
    { name = "BaseAddress",      type = "*mut PVOID", direction = "inout" },
    { name = "ZeroBits",         type = "ULONG_PTR",  direction = "in" },
    { name = "RegionSize",       type = "*mut SIZE_T", direction = "inout" },
    { name = "AllocationType",   type = "u32",        direction = "in" },
    { name = "Protect",          type = "u32",        direction = "in" },
]
# Computed at generation time
rsc_hash = 0xDEADBEEF
# Availability
available_since = "WIN_7"
category = "memory"              # inferred from name or overrides
excluded = false

[[syscall]]
name = "NtClose"
sources = ["auto", "phnt"]
ssn_x64 = 15
ssn_x86 = 15
arity_x64 = 1
arity_x86 = 1
return_type = "NTSTATUS"
params = [
    { name = "Handle", type = "HANDLE", direction = "in" },
]
rsc_hash = 0xABCD1234
category = "object"
excluded = false

# Function with opaque signature (auto but no phnt)
[[syscall]]
name = "NtUndocumentedThing"
sources = ["auto"]               # phnt had no info
ssn_x64 = 0x1E2
ssn_x86 = 0x1E2
arity_x64 = 3
arity_x86 = 3
return_type = "NTSTATUS"
# Opaque params — no types, just count
params = [
    { name = "arg0", type = "*mut c_void", direction = "inout" },
    { name = "arg1", type = "*mut c_void", direction = "inout" },
    { name = "arg2", type = "*mut c_void", direction = "inout" },
]
rsc_hash = 0x12345678
category = "other"
excluded = false
opaque_signature = true          # flag for codegen

# ...
```

### Правила приоритета при merge

```
Для каждой функции (по имени):

1. SSN / arity / RVA / stub:
   auto wins > overrides (fix_ssn, fix_arity) > else auto

2. return_type / params:
   overrides (fix_signature, add) wins > phnt > opaque fallback

3. Наличие функции:
   - add override → include
   - exclude override → exclude (excluded = true)
   - auto contains it → include
   - only phnt has it → WARN, exclude (нет в реальном ntdll)

4. rsc_hash:
   - Computed at merge time from name + RSC_SEED
   - Never overridden

5. category:
   - Inferred from name prefix / keyword
   - Can be overridden via overrides
```

### Обработка конфликтов

| Ситуация | Действие |
|---|---|
| auto: SSN=24, overrides fix_ssn=25 | overrides wins, WARN в лог |
| phnt: arity=6, overrides fix_arity=7 | overrides wins, WARN в лог |
| auto отсутствует, phnt есть | `excluded = true`, WARN "phnt-only" |
| auto есть, phnt отсутствует | `opaque_signature = true`, INFO |
| overrides add для существующей функции | ERROR — требует merge_strategy в override |

## 5. Schema versioning

- `schema_version` в каждом файле — integer, начинается с 1
- Изменение схемы = bump версии + migration script в `crates/rsc-cli/src/migrate.rs`
- Несовместимая версия → `rsc` отказывается работать, требует migration

## 6. Валидация

`rsc verify` выполняет:
- [x] Все auto/*.toml парсятся без ошибок
- [x] phnt.toml актуален относительно submodule commit
- [x] Все overrides ссылаются на существующие функции (кроме `kind=add`)
- [x] Нет дубликатов `[[syscall]] name` в merged result
- [x] Все SSN в валидном диапазоне (0x0000..0x0FFF для NT)
- [x] Все arity в разумном диапазоне (0..15)
- [x] rsc_hash — нет коллизий (если есть — ERROR)

## 8. Примеры типичных операций

### Добавить override для исправления bug в phnt

1. Редактировать `db/overrides.toml`, добавить `[[override]]` с `kind = "fix_signature"`
2. `cargo run -p rsc-cli -- merge`
3. `cargo run -p rsc-cli -- verify`
4. `cargo test --workspace`
5. Commit

### Собрать canonical для нового Windows билда

1. Запустить на target Windows: `cargo run -p rsc-collector`
2. Зафиксировать `db/auto/<new-build>.toml` в git
3. `cargo run -p rsc-cli -- diff <old> <new>` — посмотреть изменения
4. Опционально: обновить baseline в merge: `cargo run -p rsc-cli -- merge --baseline <new>`
5. Verify + test + commit

### Обновить phnt

1. `cd vendor/phnt && git pull`
2. `cargo run -p rsc-types` — регенерировать `db/phnt/phnt.toml`
3. `cargo run -p rsc-cli -- verify` — проверить что не сломалось
4. `cargo run -p rsc-cli -- merge`
5. Проверить diff canonical.toml перед коммитом — нет ли неожиданных изменений
6. Commit `vendor/phnt` pointer + `db/phnt/phnt.toml`
