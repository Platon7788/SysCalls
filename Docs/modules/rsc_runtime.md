# rsc-runtime

## Назначение

Основная runtime-библиотека (`#![no_std]`, `#![forbid(unsafe_op_in_unsafe_fn)]`).
Это то, что использует конечный потребитель — Rust-код или C через `rsc-c`.

**ZERO runtime dependencies** (hard invariant, см. maintainer dependencies doc + `DECISIONS.md` D-17).

## Статус

✅ **Phase 1 завершена** (sessions 003, 2026-04-24). Foundation готова: PEB,
hash, lazy-init syscall table с live resolve через реальный ntdll. Syscall
стабы (macro-expanded) появятся в Phase 2; autogenerated список из
canonical.toml — в Phase 6.

## Структура модулей

```
src/
├── lib.rs          facade + re-exports (~55 строк)
├── types.rs        HANDLE/PVOID/NTSTATUS/UNICODE_STRING/... (~112 строк)
├── constants.rs    STATUS_*, PAGE_*, MEM_*, PROCESS_*, THREAD_*, OBJ_* (~94 строки)
├── error.rs        NtStatus + RscResult + NtStatusExt + severity decoding (~236 строк, 9 тестов)
├── hash.rs         const fn rsc_hash (~140 строк, 5 тестов)
├── pe.rs           IMAGE_DOS/NT/EXPORT_DIRECTORY + find_export_directory (~224 строки)
├── peb.rs          get_peb (gs:[0x60] / fs:[0x30]) + find_ntdll (~193 строки)
├── table.rs        SyscallEntry + AtomicBool+UnsafeCell lazy init + resolve + pick_random_slide (~318 строк, 4 теста)
├── jumper.rs       HalosGate find_syscall_slide + xorshift32 PRNG (~180 строк, 3 теста)
└── syscalls.rs     include!() generated stubs from OUT_DIR/syscalls_generated.rs
```

Naked-стабы (x64 + x86/WoW64) материализуются `rsc-codegen::rsc_syscall!`
при сборке — см. `crates/rsc-codegen/src/lib.rs::expand_stub`.

**Всего:** ~1700 строк. Тестов: 21 (все зелёные), включая live-тесты на
реальном ntdll текущего процесса (`live_resolve_nt_close`, `live_count_reasonable`).

## Ключевые инварианты

### Hash
- `const fn rsc_hash(name: &[u8]) -> u32` — compile-time
- **`RSC_SEED = 0x52534300`** (ASCII `"RSC\0"`, см. DECISIONS D-06)
- Алгоритм: WORD-overlapping ROR8 XOR-аккумулятор (адаптирован из legacy SW3)
- Seed pinned регрессионным тестом

### PEB walking
- **x64**: `gs:[0x60]` — inline asm
- **x86**: `fs:[0x30]` — inline asm
- Обход `InLoadOrderModuleList` (первое поле = `InLoadOrderLinks` → cast напрямую)
- Case-insensitive UTF-16 match с `"ntdll.dll"`
- Bounded loop (≤ 1024 итераций) против corrupt Ldr

### Syscall table
- `MAX_SYSCALLS = 768` (запас на рост с текущих ~520)
- Entry layout: 24 bytes x64, 16 bytes x86 — зафиксировано тестом
- `filter "Zw*"` в ntdll exports → **hash считается от `"Nt..."`** (замена первых двух байт)
- Sort по `fn_addr` → индекс в sorted array = SSN
- Lazy init через `AtomicBool` (Acquire на read, Release на publish) — см. SKILLS §S-02
- Идемпотентная populate — race-safe без std::Once

### JUMPER
- **x64**: `DISTANCE_TO_SYSCALL = 0x12`, `SYSCALL_TAIL = [0x0F, 0x05, 0xC3]` (syscall; ret)
- **x86**: `DISTANCE_TO_SYSCALL = 0x0F`, `SYSCALL_TAIL = [0x0F, 0x34, 0xC3]` (sysenter; ret)
- HalosGate: 0x20-aligned поиск ±512 итераций при hook detect
- xorshift32 PRNG, seeded lazily from `rdtsc`, для per-call random pick

## Публичный API

```rust
// Hash
pub use hash::{rsc_hash, RSC_SEED};

// Resolver
pub use table::{resolve, pick_random_slide, count};

// JUMPER
pub use jumper::next_random;

// Errors
pub use error::{NtStatus, NtStatusExt, RscResult};

// Modules
pub mod constants;  // STATUS_*, PAGE_*, MEM_*, ...
pub mod error;      // NtStatus, severity
pub mod types;      // HANDLE, PVOID, ...
```

## MSRV
Rust 1.88+ (для стабильного `#[unsafe(naked)]` и `core::arch::naked_asm!`).

## Features

Нет. v1.0 намеренно без cargo-features — JUMPER через случайный slide,
essential NTSTATUS-имена и стабильный `RSC_SEED` всегда включены.

## Тесты (21 ok)

- **hash** (5): `empty_returns_seed`, `single_byte`, `no_collisions_among_representatives` (50+ NT имён), `compile_time_evaluation`, `seed_pinned`
- **error** (9): severity классов, name roundtrip, ext trait, transparent size
- **pe / peb** (implicit через table): live walk работает
- **jumper** (3): константы, tail match, PRNG variability
- **table** (4): entry layout size pinned, insertion sort, `live_resolve_nt_close` на живом ntdll, `live_count_reasonable` (≥ 100 entries)
